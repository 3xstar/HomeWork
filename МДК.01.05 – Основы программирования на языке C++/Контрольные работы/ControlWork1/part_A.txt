1. std::vector vs массив:
   · Вектор: Динамический (растёт сам), хранит размер, удаляет память сам. Живёт в куче (обычно).
   · Массив: Статический (фиксированный размер), быстрее, но сам не расширяется. Живёт на стеке (если локальный).

2. std::string_view vs std::string:
   · string_view: Только читает чужие данные. Не владеет памятью, копируется дёшево.
   · string: Владеет строкой, может её менять. Используй, если нужно сохранить или изменить данные.

3. system_clock vs steady_clock:
   · system_clock: Показывает реальное время (как на часах). Можно перевести в календарь, но его могут перевести назад (зима/лето).
   · steady_clock: Секундомер. Только идёт вперёд, идеален для замеров времени.

4. throw vs std::optional:
   · throw: Для исключительных ситуаций (ошибка парсинга, нет файла, не хватило памяти) — когда функцию дальше вызывать бессмысленно.
   · optional: Для ожидаемого отсутствия значения.

5. Этапы компиляции C++:
   1. Препроцессинг: Склейка файлов по #include, вставка макросов (#define).
   2. Компиляция: Каждый .cpp файл превращается в объектный файл — машинный код с вызовами функций из других файлов.
   3. Линковка: Сборщик (линкер) связывает все объектные файлы и библиотеки в один исполняемый файл (.exe).
